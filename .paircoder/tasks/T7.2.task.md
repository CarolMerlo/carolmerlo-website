# Objective

The About page `ld+json` schema has different social profile URLs than the homepage schema and footer. Google uses `sameAs` to link Carol's identity across platforms — inconsistent values weaken entity recognition.

# Implementation Plan

- Open `about/index.html`
- Find `<script type="application/ld+json">` in `<head>`
- Replace the `sameAs` array with the canonical set (matching homepage exactly):

```json
"sameAs": [
  "https://www.facebook.com/CarolMerloCoaching/",
  "https://www.linkedin.com/in/carolmerlo/",
  "https://www.youtube.com/@carolmerlocoaching",
  "https://www.instagram.com/carolmerlocoaching"
]
```

- No other changes to the About page

# Acceptance Criteria

- [ ] About page `sameAs` array is identical to homepage `sameAs` array
- [ ] All 4 URLs match exactly (casing, trailing slash)
- [ ] No other content on the About page is changed
- [ ] JSON is valid (no syntax errors)

# Verification

Open browser devtools → Sources → `about/index.html` → find ld+json block and confirm `sameAs` matches homepage.
