# Objective

Set up Decap CMS so Carol can update text, testimonials, and book descriptions herself via the browser without touching code. This is a meaningful architectural setup step. Confirm son's comfort with this before starting (confirmed in T1.1).

**Depends on:** T1.4–T1.12 (all pages built — content architecture in /content/ already wired)
**Launch gate:** Carol must successfully edit one testimonial and publish before site goes live.

# Implementation Plan

## 1. admin/index.html
Standard Decap CMS CDN boilerplate:
```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Content Manager</title>
</head>
<body>
  <script src="https://unpkg.com/decap-cms@^3.0.0/dist/decap-cms.js"></script>
</body>
</html>
```

## 2. admin/config.yml
Configure backend and collections:

```yaml
backend:
  name: git-gateway
  branch: main

media_folder: assets/images
public_folder: /assets/images

collections:
  - name: testimonials
    label: Testimonials
    folder: content/testimonials
    create: true
    slug: "{{fields.name | slug}}"
    fields:
      - {label: Quote, name: quote, widget: text}
      - {label: Name, name: name, widget: string}
      - {label: Context, name: context, widget: string, required: false}

  - name: books
    label: Books
    folder: content/books
    create: false
    fields:
      - {label: Title, name: title, widget: string}
      - {label: Description, name: description, widget: text}
      - {label: Amazon URL, name: amazon_url, widget: string}
      - {label: Cover Image, name: cover_image, widget: image}

  - name: pages
    label: Page Content
    files:
      - name: homepage
        label: Homepage
        file: content/pages/homepage.json
        fields:
          - {label: My Books Blurb, name: books_blurb, widget: text}
      - name: pricing
        label: Pricing
        file: content/pricing.json
        fields:
          - {label: Pay in Full Price, name: pay_full, widget: string}
          - {label: Payment Plan Price, name: pay_plan, widget: string}
          - {label: What's Included, name: whats_included, widget: list, field: {name: item, widget: string}}
```

## 3. Netlify Identity
1. Enable Netlify Identity in Netlify dashboard (Site Settings → Identity → Enable Identity)
2. Set registration: Invite only
3. Enable Git Gateway (Identity → Services → Git Gateway)
4. Invite Carol as editor user

## 4. Content files
Create initial JSON/Markdown files in /content/ with current content already populated:
- `content/testimonials/[name].json` — one per testimonial (populate all testimonials from homepage, about, spiritual coaching)
- `content/books/[slug].json` — one per book (5 books)
- `content/pages/homepage.json` — homepage book blurb
- `content/pricing.json` — pricing data

## 5. Launch Gate Test
Carol must:
1. Go to `[preview-url]/admin`
2. Log in with her invited credentials
3. Edit one testimonial (change a word)
4. Click Publish
5. Verify the change appears on the live site after Netlify rebuild (1–2 min)

This test must pass before DNS cutover (T1.15).
